Log!t
Technical Architecture & Design

A premium personal movie logger. Engineered with vanilla client-side web technologies, integrating Supabase DB, Google Drive Backup Engines, and TMDB REST APIs.

CONCEPT Overview

System Concept & Architectural Goals

System Value

Log!t is a high-performance personal library tracking movie logs, watch histories, ratings, and stats metrics.

Built serverless and client-focused, it removes host-dependency using custom API bridges directly from the user's browser context.

Architectural Pillars

  • Supabase Cloud — Low-latency primary storage and authentication database.
  • Google Drive Integration — Fully secure user-owned data storage backups.
  • Zero Compile Pipeline — Pure vanilla modules loading dynamically with no bundler bloat.
Google Drive API

Re-engineered Google Drive Engine

Modern Fetch Pipeline

  • Async/Await Wrapper — Purged legacy XHR codes for streamlined Fetch APIs.
  • Token Auto-Expiry — Automatic detection of 401 error codes, clearing expired caches, and prompting reconnect states.
  • User Info Resolution — Calls userinfo endpoint with clean API-About fallbacks.

Payload & Queries

  • RFC 2046 Multipart Body — Formats file uploads using related boundary markers to deliver metadata & content in single payload.
  • Query Isolations — Excludes trashed files using Google Drive Query DSL.
  • Dynamic Naming — Generates file names with backup counts and timestamps.
Data Operations

End-to-End Data Pipeline

Movie Management Flow

Search movie triggers real-time queries against TMDB API. Storing details caches locally and uploads to Supabase cloud tables.

Any movie card mutations instantly queue dynamic Google Drive auto-backups in background task loops.

Data Integrity Indicators

  • Incomplete Audit Engine — Scans collection for missing ratings or runtime files.
  • Diagnostic Overlays — Displays visual indicators (red borders and dot markers) for items with incomplete metadata.
  • Month Headers — Propagates status notifications upstream to summary groups.
User Interface

Refined Minimalism & Grid Spacing

Apple-Inspired Layout

  • 1-Row Cloud Status Card — Clean compact bar with dynamic badge signals.
  • Sleek Grouped Sections — Reorganized settings lists into structured Account, Drive, Backup, and Danger blocks.
  • Absolute Centering — Profile headers center username to exact pixel bounds.

Adaptive Grid Flow

  • 24px Section Margins — Standardized mobile vertical gutters between cards.
  • Chevron Indicators — Uniform navigation chevrons indicating click pathways.
  • Pulsing Connections — CSS animations visualize active token verification.
